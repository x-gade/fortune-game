# Fortune Game

## О проекте

**Fortune Game** - это настольное игровое приложение на Python для проведения интеллектуальной викторины в формате, близком к «Что? Где? Когда?» с механикой колеса фортуны.

Проект предназначен для ведущего игры и решает следующие задачи:

* управление командами и их очками;
* управление раундами;
* хранение и выбор вопросов;
* индивидуальные таймеры для каждого вопроса;
* визуальное колесо фортуны;
* поддержка видеовопросов;
* автоматическое исключение уже сыгранных вопросов;
* единый интерфейс управления игрой.

На текущем этапе реализуется MVP с упором на архитектуру, а не на визуал.

---

## Ключевая игровая механика

Основной принцип:

**колесо не выбирает вопрос — вопрос выбирается заранее, а колесо лишь визуализирует этот выбор.**

Жизненный цикл хода:

1. ведущий нажимает "Крутить";
2. система выбирает случайный доступный вопрос;
3. колесо анимируется и останавливается на этом вопросе;
4. вопрос удаляется из пула доступных;
5. если есть видео - сначала проигрывается видео;
6. после видео запускается таймер;
7. ведущий фиксирует результат;
8. начисляются очки.

## Архитектура проекта

Проект разделён на 3 слоя:

### 1. Data

* `data/`
* JSON-файлы

### 2. Logic

* `models/`
* `services/`

### 3. UI (будущее)

* `ui/`

Это позволяет:

* менять интерфейс без переписывания логики;
* масштабировать проект;
* избежать хаоса в коде.

---

## data/

Хранит игровые данные.

### game_data.json

Главный источник данных MVP.

Содержит:

* команды
* раунды
* вопросы
* настройки

### Расширенная структура вопроса

```json
{
  "id": 1,
  "round_id": 1,
  "team_id": null,
  "text": "Вопрос",
  "answer": "Ответ",
  "timer_seconds": 20,
  "points": 100,
  "used": false,
  "media_type": "video",
  "media_path": "data/media/video_secret_1.mp4"
}
```

### media_type

* null - обычный вопрос
* video - видеовопрос

## Видеовопросы

Поддержка реализуется через JSON.

Логика:

1. если media_type == video
2. открывается плеер
3. проигрывается видео
4. после окончания стартует таймер

---

## Работа колеса

Правильная модель:

* список секторов = список доступных вопросов
* после использования вопрос удаляется
* колесо уменьшается

Это обеспечивает:

* честную игру
* контроль логики
* отсутствие повторов
