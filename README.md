# Quiz Manager

## О проекте

Quiz Manager это desktop-приложение на Python для проведения интеллектуальной викторины с визуальным колесом вопросов, двухоконным интерфейсом и поддержкой видеовопросов.

Проект ориентирован на реальное проведение игр ведущим. В текущем состоянии это уже не концепт и не консольный MVP, а рабочее GUI-приложение на PySide6, в котором реализован основной игровой цикл, управление вопросами, счетом, таймером и видео.

Приложение разделено на два окна:

* окно администратора для управления ходом игры
* публичное окно для игроков и зрителей

Система позволяет проводить игру в формате, близком к Что? Где? Когда?, с отдельным экраном вывода, визуальным колесом, таймером, видео и сохранением состояния вопросов.

На текущем этапе реализовано:

* полноценное GUI-приложение на PySide6

* выбор раундов и управление очередью команд
* визуальное колесо с анимацией и фиксированной остановкой на заранее выбранном вопросе
* поддержка текстовых вопросов, видеовопросов и видеоответов
* таймер с запуском, паузой, продолжением и остановкой
* управление воспроизведением видео с паузой и продолжением
* повторный показ вопроса и повторное воспроизведение видео вопроса

## Основная игровая логика

Ключевой принцип проекта:

вопрос выбирается логикой заранее, а колесо не определяет вопрос, а только визуализирует уже сделанный выбор.

Жизненный цикл одного хода:

1. ведущий выбирает раунд
2. система определяет текущую отвечающую команду по очереди
3. ведущий нажимает кнопку запуска колеса
4. система выбирает случайный доступный вопрос текущего раунда
5. колесо анимируется и останавливается на выбранном секторе
6. вопрос показывается в админском и публичном окне
7. если вопрос видеотипа, запускается видео вопроса
8. после завершения видео вопроса автоматически запускается таймер
9. ведущий может поставить таймер на паузу, продолжить его или остановить
10. ведущий может показать ответ вручную
11. ведущий фиксирует результат кнопкой Верный ответ или Неверный ответ
12. очки начисляются только при верном ответе
13. вопрос помечается как использованный и сохраняется
14. очередь переходит к следующей команде

## Архитектура

Проект построен по разделению ответственности между слоями:

```text
UI -> Controller -> Services -> Models -> Data
```

### Data

Слой хранения игровых данных.

Используется JSON-файл `data/game_data.json`, в котором хранятся:

* настройки
* команды
* раунды
* вопросы

### Models

Описание сущностей предметной области.

* `models/team.py` - команда и ее счет
* `models/round.py` - раунд игры
* `models/question.py` - вопрос, ответ, очки, таймер, статус, медиа
* `models/game_state.py` - runtime-состояние игры

### Services

Бизнес-логика и работа с данными.

* `services/data_loader.py` - загрузка и сохранение JSON
* `services/question_service.py` - выбор, фильтрация, сброс и обновление вопросов
* `services/score_service.py` - начисление и списание очков
* `services/wheel_service.py` - формирование подписей колеса и поиск целевого сектора
* `services/timer_service.py` - консольный таймер из ранней версии проекта
* `services/game_service.py` - логика CLI-версии и агрегация сервисов

### UI

Графический интерфейс на PySide6.

* `ui/app.py` - основной UI-контроллер, координация окон, таймера, видео, очереди команд и состояния игры
* `ui/admin_window.py` - окно ведущего и весь управляющий интерфейс
* `ui/display_window.py` - публичное окно для вывода игры
* `ui/wheel_widget.py` - кастомный виджет колеса с анимацией
* `ui/timer_widget.py` - визуальный виджет таймера
* `ui/scoreboard_widget.py` - табло счета команд
* `ui/video_widget.py` - воспроизведение видео
* `ui/question_editor_dialog.py` - диалог создания нового вопроса

### Utils

Вспомогательные функции проекта.

* `utils/validators.py` - валидация структуры JSON
* `utils/formatters.py` - форматирование данных для CLI-части
* `utils/helpers.py` - простые вспомогательные утилиты

## Формат данных

Основной источник данных проекта это `data/game_data.json`.

В файле находятся:

* `settings`
* `teams`
* `rounds`
* `questions`

Пример структуры вопроса:

```json
{
  "id": 1,
  "round_id": 1,
  "team_id": null,
  "text": "Вопрос",
  "answer": "Ответ",
  "timer_seconds": 30,
  "points": 100,
  "used": false,
  "category": null,
  "difficulty": null,
  "media_type": "video",
  "media_path": "data/media/question_1"
}
```

### Поля вопроса

* `id` - уникальный идентификатор
* `round_id` - привязка к раунду
* `team_id` - резерв под привязку к команде
* `text` - текст вопроса
* `answer` - текст ответа
* `timer_seconds` - длительность таймера
* `points` - количество очков
* `used` - был ли вопрос уже сыгран
* `category` - категория вопроса
* `difficulty` - уровень сложности
* `media_type` - тип медиа
* `media_path` - путь к папке медиа

### Типы вопросов

Поддерживаются:

* обычный текстовый вопрос
* видеовопрос

Если `media_type == "video"`, ожидается папка `media_path`, внутри которой могут находиться:

* `video_secret.mp4` - видео вопроса
* `video_response.mp4` - видео ответа

Если `video_response.mp4` отсутствует, ответ может быть текстовым.

## Логика видео

В проекте реализована отдельная логика для видеовопросов и видеоответов.

### Видеовопрос

* после завершения вращения колеса показывается вопрос
* если вопрос видеотипа, запускается `video_secret.mp4`
* после окончания видео автоматически стартует таймер
* кнопку повтора вопроса можно использовать для повторного запуска видео вопроса

### Видеоответ

* при нажатии `Показать ответ` запускается `video_response.mp4`, если он существует
* если у вопроса есть текст ответа, он отображается в админском окне
* при повторном нажатии `Показать ответ` видеоответ запускается заново

### Защита от повторного показа ответа

Если ведущий уже нажал `Показать ответ`, то последующее нажатие `Верный ответ` или `Неверный ответ` не должно повторно воспроизводить ответ.

Кнопки результата в этом случае выполняют только логику фиксации исхода:

* `Верный ответ` начисляет очки
* `Неверный ответ` не начисляет очки

### Пауза и продолжение видео

Для видео реализовано:

* пауза
* продолжение
* динамическая кнопка в админке

Кнопка управления видео появляется только если текущий контекст относится к видеовопросу и видео реально отображается на публичном окне.

Состояния кнопки:

* `Пауза видео`
* `Продолжить видео`

## Логика таймера

В GUI-версии таймер управляется через `GameController`.

Поддерживаются действия:

* запуск
* пауза
* продолжение
* остановка и сброс

### Текущее поведение

* если после запуска прошло некоторое время, таймер уменьшает `remaining_seconds`
* при паузе значение сохраняется
* при продолжении отсчет идет с сохраненного значения
* при стопе значение сбрасывается к исходному времени вопроса
* после окончания видео вопроса таймер может запускаться автоматически

### Кнопка паузы и продолжения

В админке реализована одна переключаемая кнопка:

* во время отсчета: `Пауза таймера`
* в режиме паузы: `Продолжить таймер`

Это поведение синхронизировано с визуальным виджетом таймера.

## Управление вопросами в интерфейсе

Из админского окна можно:

* просмотреть все вопросы
* фильтровать по раунду
* увидеть статус вопроса
* вручную менять `used`
* сбросить один вопрос
* сбросить все вопросы раунда
* сбросить все вопросы проекта
* создать новый вопрос

Список вопросов пересобирается динамически после изменений.

---

## Добавление новых вопросов

Для новых вопросов используется отдельный диалог.

Поддерживаются:

* текстовые вопросы
* видеовопросы

Для видеовопроса можно выбрать папку из `data/media`. Диалог проверяет:

* существует ли папка
* есть ли `video_secret.mp4`
* нужен ли текстовый ответ, если нет `video_response.mp4`

Это уменьшает вероятность некорректных данных в JSON.

---

## Сохранение состояния

После изменений игра сохраняет данные обратно в `data/game_data.json`.

Сохраняются:

* команды и их очки
* список вопросов
* статусы `used`
* добавленные вопросы

Это позволяет продолжить работу без ручного восстановления состояния.
